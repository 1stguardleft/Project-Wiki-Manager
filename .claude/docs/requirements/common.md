# Requirements

프로젝트 wiki-manager의 요구사항 정의서.
[llm-wiki.md](references/llm-wiki.md) 문서를 참고하여 작성.

---

## 개요

LLM이 직접 유지·관리하는 개인 지식 베이스(wiki) 시스템.
RAG처럼 쿼리마다 원본 문서를 다시 검색하는 방식이 아니라, 소스를 추가할 때마다 LLM이 wiki를 점진적으로 갱신하고 합성하는 방식으로 동작한다.

---

## 핵심 개념

- **누적(Compounding) 지식**: 새 소스를 추가하면 LLM이 기존 wiki를 읽고 통합·갱신한다. 지식은 매 쿼리마다 재발견되는 것이 아니라 축적된다.
- **LLM이 wiki를 작성**: 사용자는 소스 수집과 탐색·질문을 담당하고, 요약·상호 참조·색인·유지보수는 LLM이 수행한다.
- **Obsidian 연동**: 사용자는 Obsidian으로 wiki를 열람하고, LLM은 파일을 직접 편집한다.

---

## 아키텍처 (3-레이어)

| 레이어 | 설명 | 소유자 |
|--------|------|--------|
| **Raw Sources** | 원본 문서 컬렉션 (기사, 논문, 이미지 등). 불변. | 사용자 |
| **Wiki** | LLM이 생성·유지하는 마크다운 파일 디렉토리. 요약, 엔티티 페이지, 개념 페이지, 비교, 개요, 합성 등. | LLM |
| **Schema** | LLM에게 wiki 구조, 관례, 워크플로를 지시하는 설정 문서 (CLAUDE.md). | 사용자 + LLM 공동 진화 |

## 기능 요구사항

### 1. Ingest (소스 수집)

- 사용자가 raw 소스를 추가하고 LLM에 처리를 요청한다.
- LLM은 다음 흐름을 수행한다:
  1. 소스 읽기
  2. 사용자와 핵심 내용 논의
  3. wiki에 요약 페이지 생성
  4. `index.md` 갱신
  5. 관련 엔티티·개념 페이지 갱신 (소스 하나당 10~15개 페이지 영향 가능)
  6. `log.md`에 항목 추가
- 소스 1개씩 순차 처리 또는 배치 처리 모두 지원.

### 2. Query (질의응답)

- 사용자가 wiki에 대해 질문하면 LLM이 관련 페이지를 검색·읽고 인용과 함께 답변을 합성한다.
- 답변 형식: 마크다운 페이지, 비교 테이블, 슬라이드(Marp), 차트(matplotlib) 등.
- 가치 있는 답변(비교, 분석, 발견된 연결)은 wiki에 새 페이지로 저장한다.

### 3. Lint (wiki 건강 점검)

- LLM이 주기적으로 wiki를 점검한다:
  - 페이지 간 모순 탐지
  - 새로운 소스에 의해 무효화된 오래된 주장
  - 인바운드 링크가 없는 고아 페이지
  - 페이지가 없는 중요 개념
  - 누락된 상호 참조
  - 웹 검색으로 채울 수 있는 데이터 공백
- 새로운 조사 질문 및 소스 추천 기능 포함.

## 인덱싱 및 로깅

### `index.md`

- 역할: 콘텐츠 지향 카탈로그.
- 형식: 카테고리별(엔티티, 개념, 소스 등) 페이지 목록 + 링크 + 한 줄 요약 + 선택적 메타데이터(날짜, 소스 수).
- 갱신 시점: 매 ingest 시.
- 용도: LLM이 쿼리 답변 시 index를 먼저 읽고 관련 페이지를 특정한다.

### `log.md`

- 역할: 시간순 추가 전용(append-only) 기록.
- 항목 예시: ingest, query, lint 작업.
- 항목 형식: `## [YYYY-MM-DD] <type> | <title>` (unix 도구로 파싱 가능).
- 용도: wiki 진화 타임라인 제공, LLM이 최근 작업 파악에 활용.

## 비기능 요구사항

- **불변 소스**: raw 소스는 LLM이 절대 수정하지 않는다.
- **마크다운 기반**: wiki 전체가 마크다운 파일로 구성되어 git으로 버전 관리 가능.
- **로컬 우선**: 이미지 등 첨부 파일은 로컬에 저장.
- **확장성**: 소규모(수십 소스)에서는 index.md만으로 충분하며, 규모 증가 시 검색 엔진(qmd 등) 도입 가능.

---

## 선택적 기능 (Optional)

- **CLI 검색 도구**: wiki 페이지 대상 로컬 검색 (BM25/벡터 하이브리드, e.g. qmd).
- **MCP 서버 통합**: LLM이 검색 도구를 네이티브 툴로 사용.
- **Obsidian 플러그인**:
  - Web Clipper: 웹 기사를 마크다운으로 변환.
  - Marp: wiki 콘텐츠로 슬라이드 생성.
  - Dataview: YAML 프론트매터 기반 동적 테이블/목록.
- **이미지 로컬 저장**: 첨부 파일 자동 다운로드.

## 적용 사례 (Use Cases)

| 도메인 | 설명 |
|--------|------|
| 개인 | 목표, 건강, 자기계발 — 일기, 기사, 팟캐스트 메모 통합 |
| 리서치 | 논문·기사·보고서를 수주~수개월에 걸쳐 축적, 진화하는 테제 관리 |
| 독서 | 챕터별 파일링, 인물·테마·플롯 스레드 페이지 구축 |
| 팀/비즈니스 | Slack 스레드, 회의록, 프로젝트 문서로 내부 wiki 유지 |
| 기타 | 경쟁 분석, 실사, 여행 계획, 강의 노트, 취미 심화 학습 등 |
